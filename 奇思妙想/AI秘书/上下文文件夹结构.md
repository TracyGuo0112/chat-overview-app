## 上下文文件夹结构

### 1. 基础配置文件夹
```
/config
  ├── agents/                  # 各agent配置文件
  │   ├── news_reader.yaml     # 新闻播报员配置
  │   ├── product_researcher.yaml  # 产品情报员配置
  │   ├── reading_assistant.yaml   # 阅读助手配置
  │   ├── scheduler.yaml       # 日程安排员配置
  │   └── daily_summary.yaml   # 当日总结员配置
  ├── integrations/            # 第三方集成配置
  │   ├── feishu.yaml          # 飞书API配置
  │   ├── social_media.yaml    # 社交媒体平台配置
  │   └── websites.yaml        # 网站监控配置
  └── schedules/               # 定时任务配置
      ├── daily_news.yaml      # 每日新闻抓取时间
      ├── weekly_report.yaml   # 每周报告生成时间
      └── daily_summary.yaml   # 每日总结时间
```

### 2. 数据存储文件夹
```
/data
  ├── news/                    # 新闻数据
  │   ├── raw/                 # 原始抓取数据
  │   ├── processed/           # 处理后的新闻
  │   └── summaries/           # 新闻摘要
  ├── products/                # 产品情报数据
  │   ├── competitor_analysis/ # 竞品分析
  │   ├── user_feedback/       # 用户反馈
  │   └── trend_reports/       # 趋势报告
  ├── reading/                 # 阅读相关数据
  │   ├── unread_docs/         # 未读文档列表
  │   ├── reading_progress/    # 阅读进度
  │   └── recommendations/     # 推荐内容
  ├── schedule/                # 日程数据
  │   ├── daily_plans/         # 每日计划
  │   └── task_history/        # 任务历史
  └── summaries/               # 总结数据
      ├── daily/               # 每日总结
      └── weekly/              # 每周总结
```

### 3. 日志文件夹
```
/logs
  ├── agents/                  # 各agent运行日志
  │   ├── news_reader.log
  │   ├── product_researcher.log
  │   ├── reading_assistant.log
  │   ├── scheduler.log
  │   └── daily_summary.log
  ├── errors/                  # 错误日志
  └── api/                     # API调用日志
      ├── feishu_api.log
      └── social_media_api.log
```

### 4. 文档输出文件夹
```
/docs
  ├── news_briefs/             # 新闻简报
  ├── product_reports/         # 产品报告
  ├── reading_guides/          # 阅读指南
  ├── daily_plans/             # 每日计划
  └── daily_summaries/         # 每日总结
```

### 5. 脚本文件夹
```
/scripts
  ├── crawlers/                # 数据爬虫脚本
  ├── processors/              # 数据处理脚本
  ├── generators/              # 报告生成脚本
  └── utilities/               # 工具脚本
```
